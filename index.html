Használat

Pull Request

A lehívási kérések segítségével elmondhatja másoknak a GitHub lerakatában lévő repository-hoz telepített változásokat.
A lekérési kérelem megnyitása után megbeszélheti és áttekintheti a lehetséges változásokat az együttműködőkkel,
és további kötelezettségvállalásokat adhat hozzá, mielőtt a módosításokat egyesíti az alapággal.

Push

Alapértelmezés szerint a git push csak a megfelelő ágat frissíti, tehát ha a git push végrehajtásakor kijelentkezik a master ágra,
akkor csak a master ág frissül. Mindig jó ötlet a git státusz használatával megnézni, hogy milyen ágon van, mielőtt a feltölti.
